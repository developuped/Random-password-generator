# Study Planner — Практична №1

**Короткий пітч (2–3 речення)**
Study Planner допомагає студентам планувати навчальні сесії, контролювати задачі та відмічати виконане. Дозволяє швидко зафіксувати коротку задачу, побачити її в списку і відзначити як виконану — цього достатньо для перевірки гіпотези цінності.

**Проблема (у термінах користувача)**
Студенти втрачають фокус і забувають про важливі навчальні завдання через відсутність простої системи планування й відстеження прогресу.

**Мета першої ітерації**
Створити мінімально життєздатний продукт (MVP), у якому можна:
- додати задачу;
- побачити список задач;
- відзначити задачу як виконану.

**MVP (ядро цінності)**
1. Форма додавання нової задачі (текст + опціонально термін);
2. Список задач (сортування за датою або статусом);
3. Позначення задачі як виконаної;
4. Збереження даних локально (файлом або у браузерному localStorage).

**Коротка карта стейкхолдерів**
- Користувачі: студенти, які потребують простого трекера задач.
- Оцінювачі/лектори: перевіряють виконання практичної.
- Розробник(и): відповідають за реалізацію та документування.
- Зацікавлені (опціонально): дизайнери/ментори, які впливають на UX.

**Definition of Success — ознаки успіху**
- У репозиторії є README.md з пітчем, MVP та критеріями успіху.
- Користувач може виконати головний сценарій (додати → переглянути → відзначити) без інструкцій за ≤ 3 кроки.
- Три фундаментальні тести (додавання, відображення, відзначення) проходять ручну перевірку.
- Файли документів додані до репозиторію і мають адекватні коментарі до комітів.
